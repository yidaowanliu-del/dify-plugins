# 打包为本地文件并分享

`.difypkg` 是 Dify 插件的可移植形式。生成该文件后，任何人都可在插件页面上传文件来安装。这种方式适用于 **私有插件**、**内部测试**，或面向小范围群体的 **非公开分发**。

> 来源：[Dify 官方文档 - 打包为本地文件并分享](https://docs.dify.ai/zh/develop-plugin/publishing/marketplace-listing/release-by-file)

## 前提条件

- 已安装 [Dify 插件 CLI](https://docs.dify.ai/zh/develop-plugin/getting-started/cli)。运行 `dify version` 进行确认。
- 一个开发完成的插件项目。
- 打包前已通过[远程调试](https://docs.dify.ai/zh/develop-plugin/features-and-specs/plugin-types/remote-debug-a-plugin)测试插件。

> **重要**: `manifest.yaml` 以及每个 `provider/*.yaml` 中的 `author` 字段都必须与你的 GitHub 用户名一致。不一致会在安装时导致 `plugin_unique_identifier is not valid` 错误。

## 打包插件

1. 进入插件项目的 **上级** 目录。
2. 运行打包命令：

```bash
dify plugin package ./your_plugin_project
```

CLI 会在当前目录生成一个 `your_plugin_project.difypkg` 文件。

## 从文件安装

1. 在 Dify 中，点击右上角的 **插件** 图标。
2. 选择 **安装插件 > 通过本地文件**，或将 `.difypkg` 文件拖放到插件页面的空白区域。
3. 查看插件请求的权限，然后点击 **安装**。

> 自部署 Dify 默认强制执行签名验证。通过本地文件安装的未签名 `.difypkg` 文件会被拒绝，除非你为其签名或更改验证设置。参见[第三方签名验证](https://docs.dify.ai/zh/develop-plugin/publishing/standards/third-party-signature-verification)。

## 分享文件

将 `.difypkg` 文件直接交给用户：通过电子邮件、内部文件共享，或任何你能掌控的渠道。

如果希望更广泛地分发，可考虑：

- **发布到 GitHub**: 将 `.difypkg` 附加到 GitHub release，用户通过仓库 URL 安装。
- **提交到市场**: 向 `langgenius/dify-plugins` 提交 PR，进入官方目录。
